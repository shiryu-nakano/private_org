ここで **「プロジェクトノート ↔ 日次ノートをちゃんとリンクさせる仕組み」** を一気に形にしましょう。

やることはこの3つです：

1. `~/org/roam/projects/` にプロジェクトノートを作る仕組みを用意
2. org-roam のショートカット（find / insert）を入れる
3. daily の LOG からプロジェクトノートにリンクを貼る運用を決める

---

## 1️⃣ プロジェクトノート用のディレクトリ＆テンプレを作る

まずディレクトリを作ります：

```bash
mkdir -p ~/org/roam/projects
```

次に `init.el` の org-roam 設定のすぐ下あたりに **プロジェクト用 capture テンプレ**を追加します。

### 追加するコード（コピペOK）

```elisp
;; org-roam 基本設定（あなたの既存設定＋少し拡張）
(use-package org-roam
  :ensure t
  :custom
  (org-roam-directory "~/org/roam")
  :config
  (org-roam-db-autosync-mode))

;; ノード検索 / 挿入のショートカット（超重要）
(global-set-key (kbd "C-c n f") #'org-roam-node-find)
(global-set-key (kbd "C-c n i") #'org-roam-node-insert)

;; プロジェクトノート用テンプレ
(setq org-roam-capture-templates
      '(("p" "Project" plain
         "%?"
         :target (file+head "projects/${slug}.org"
                            "#+title: ${title}\n#+filetags: :project:\n\n* Overview\n\n* Tasks\n\n* Notes\n"))))
```

### これで何ができる？

1. `M-x org-roam-node-find` か `C-c n f`

2. プロンプトでプロジェクト名を入力（例：`ROS2 Planning`）

3. なければ新規作成され、ファイルは：

   ```text
   ~/org/roam/projects/ros2-planning.org
   ```

4. 中身は：

   ```org
   #+title: ROS2 Planning
   #+filetags: :project:

   * Overview

   * Tasks

   * Notes
   ```

これが「プロジェクトノート（知識の集約場所）」になります。

---

## 2️⃣ 日次ノートからプロジェクトノートへリンクを貼る

今の daily テンプレはこんな感じですよね（LOG 部分だけ抜粋）：

```org
*LOG
**14:32
   ;; ここに書き始める
```

ここから **プロジェクトノートへのリンク**を生やします。

### 基本の操作パターン（いちばんシンプル）

1. `C-c n i`（org-roam-node-insert）を押す
2. プロンプトにプロジェクト名を打つ

   * 例：`ROS2 Planning` / `SLT rLLC` など
3. 候補からプロジェクトを選ぶ

すると LOG の中にこういうリンクが入ります：

```org
*LOG
**14:32
   PurePursuit のバグ修正 [[id:abc123][ROS2 Planning]]
```

- `id:abc123` は org-roam が自動で付ける ID
- `[ROS2 Planning]` がリンクの見た目

これで：

- 日次ノート → プロジェクトノート にジャンプできる
- プロジェクトノート側からも backlinks でその日付のログが見える

---

## 3️⃣ プロジェクトノート側にも「日次ログへのリンク」を置きたい場合（任意）

必須ではないけど、やるとかなり気持ちいい運用です。

### 流れ

1. 日次ノート（`daily/2025/12/2025-12-05.org`）を開く
2. `M-x org-id-get-create` を見出しの上で実行すると、その見出しに ID がつく
3. プロジェクトノート（例：`ros2-planning.org`）を開いて、`C-c l` → `C-c C-l` でリンクを貼る

例えばプロジェクトノートに：

```org
*Notes
**2025-12-05 PurePursuit デバッグ
   [[id:xxxxxxx][2025-12-05 の作業ログ]]
   - ここに軽くまとめを書く
```

こうすると：

- プロジェクトノートから
  → その日の detailed な LOG に飛べる
- 日次ノートから
  → プロジェクトの全体像に飛べる

**行き来しやすい二重リンク構造**になります。

---

## 4️⃣ あなた向けの「現実的な運用パターン」

あまり複雑なことを一気にやるとしんどいので、まずはこれだけでOK：

### ✅ 毎朝〜作業開始

- `C-c n d` で今日の daily を開く
- `* TASK` に軽く「今日はこれやりたい」箇条書き

### ✅ 作業中（LOG）

- バグ調査や論文読むとき

  * `* LOG` → `** HH:MM` で始める
  * プロジェクト名を `C-c n i` で挿入 → リンク化
- 例：

  ```org
  * LOG
  ** 15:12
     [[id:ros2-planning][ROS2 Planning]] PurePursuit のパラメータ調整
     - 角速度制限を X → Y に変更
     - 結果：カーブでオーバーシュート減少
  ```

### ✅ プロジェクトノート側

- `C-c n f` → プロジェクト名選択
- `* Notes` に「今日のまとめ」を1〜2行だけ書く
- 詳細ログは daily 側に任せる

---

## 5️⃣ いまの状態に合わせた「整合性チェック」

あなたの現状：

- `org-roam-directory` → `~/org/roam` ✅
- dailies → `daily/%Y/%m/%Y-%m-%d.org` ✅
- まだ `projects/` ディレクトリと `org-roam-capture-templates`（project）は未整備 → いまやった ✅

なので、このあとやるべきことはシンプルです：

1. `mkdir -p ~/org/roam/projects` 済ませる
2. `init.el` に上の **org-roam-capture-templates と C-c n f / C-c n i** を追加
3. Emacs 再起動 or `M-x eval-buffer`
4. `C-c n f` → `ROS2 Planning` など打ってプロジェクトノートを1つ作る
5. `C-c n d` で daily を開き、`C-c n i` でそのプロジェクトへのリンクを LOG に挿入してみる

ここまでできたら、**Step 5：プロジェクトノートとの相互リンク** は実質完成です。

---

もしよければ：

- 一回 `C-c n f` で `ROS2` 系・`SLT` 系のプロジェクトノートを何個か作ってみて
- そのスクショやファイル名リスト（`tree ~/org/roam`）を貼ってくれれば、
  **それを前提に「プロジェクト別ダッシュボード（C）」の agenda ビューを一緒に組みます。**
